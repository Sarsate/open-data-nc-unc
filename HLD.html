Diagram

The architecture is a standard Model-View-Controller paradigm, where the view exists on the client's browser, the model is stored on the database, and the controller manages the interactions between the two with server code. The database will be Postgresql, and the Server-side framework will be Django.

Django uses python to dynamically generate content for the client using HTML templates, compressed CSS (Less), and static Javascript files. Django interfaces with the database with it's built in Object Relational Mapping. Database models and relations can be expressed as python objects in Django's models.py files, abstracting those disgusting postgres database queries.
